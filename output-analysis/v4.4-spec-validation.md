v6 is a minor-style delta on top of v5 (the spec file inherits v5
byte-for-byte for every section not enumerated below). The
numbered increments are:

- **§1.1 (Enhancement-layer transparent video overlays bullet)** —
  closes v5 detail-review flag-3 by tightening the OOS bullet.
  v5's bullet read "Composition of overlays via codec enhancement
  layer... is out of scope for this edition", which an implementer
  could read as forbidding scalable-codec carriers altogether. v6
  rewrites the bullet to explicitly: (a) prohibit defining a new
  normative construct that ties enhancement-layer Representations
  to primary base layers, and (b) admit that a Broadcaster
  authoring a `video` form (§5.3.2) whose sub-MPD carries a
  scalable-codec Representation is conformant — the Player treats
  it as a regular video form per §5.3.4 and the enhancement-layer
  semantics, if any, are out of scope. Annotation:
  `<!-- refine: v5-detail-review.md#flag-3 -->`.

- **OQ-3 "Origin in this spec" cell** — closes v5 detail-review
  flag-1 by rewording the cell. v5's cell `§4.1 three-actor model
  bypassed for non-ad content` reads as if §4.1 already includes
  a bypass path, which it does not. v6 reformulates as `§4.1
  three-actor model (the open question, if admitted, would bypass
  the ADS round-trip for non-ad concurrent content)`, making
  explicit that the bypass is the proposed change, not a current
  feature. Annotation:
  `<!-- refine: v5-detail-review.md#flag-1 -->`.

- **NEW lifecycle framing paragraph** in the
  "Open questions for WG (2026-06-02 meeting)" section — addresses
  the first stylistic note from v5 detail-review "Open questions
  surfaced" (the section's lifecycle was unspecified). v6 adds an
  explicit one-paragraph framing: the section is transient,
  removed in the major build that follows each OQ's resolution,
  with the resolution recorded as an ADR. Annotation:
  `<!-- refine: v5-detail-review.md#open-questions-lifecycle -->`.

- **Version-line bump** — `v5 -> v6` at the top metadata block.

# DASH conformance audit — v7 (2026-05-20)

**Audit target**: `../output/v7-sgai-spec.md`.
**Reference**: MPEG-DASH 6th edition (ISO/IEC 23009-1:2025, FDIS),
consulted via NotebookLM notebook "Streaming Protocols — DASH,
HLS, C2PA, DRM" — but in this run NO new NotebookLM queries were
issued. The v4.2 audit's Q1 / Q2 grounding (verbatim §5.16.5 unit
declaration; `<xs:element>` absence for `UrlParamInfo` /
`RequestParam` in the printed schema) covers every construct that
v7 carries over. v7 introduces NO new constructs.

**Method**: inventory of new / modified constructs introduced by
the spec, per-construct verdict (Conforming `V` / Marginal `M` /
Non-conforming `N`).

## Scope

v7 is a minor-style editorial iteration on top of v6. The XML
construct surface is byte-identical to v6 (no new schemes, no
new attributes, no new elements, no annex example changed). v7
only changes prose in two places:

- "Open questions for WG (2026-06-02 meeting)" section: "Origin
  in this spec" column normalisation across all twelve OQ rows
  (closes v6 detail-review flagged item #2). The "Open question"
  column itself is byte-identical to v6.
- Version-line bump (v6 -> v7).

The audit therefore carries every verdict forward unchanged from
v6 (which itself carried every verdict from v5). The construct
inventory below restates the v6 audit's C-01..C-28 with no
verdict changes. The audit registers NO new findings.

This audit is conformance-against-DASH-6th only. Design-level
audit lives in `v7-spec-validation.md`; detail review lives in
`v7-detail-review.md`.

## Method summary

- Inventory carried over from v6 audit `C-01..C-28` (the

## Open questions surfaced

- The Open-questions-for-WG section in v7 is **outside** the
  conformance audit's surface: it is informative material, and
  none of the OQ-1..OQ-12 items mutates a normative construct in
  v7. The audit does NOT carry an open question per OQ — they are
  tracked in `v7-spec-validation.md` as informative-only. The
  column-uniformity normalisation (v7 delta) is editorial inside an
  informative section.
- The conformance audit's open-question list is therefore
  unchanged from v6: the published DASH-MPD.xsd's authoritative
  declaration of `UrlParamInfo` / `RequestParam` is still outside
  the corpus available to NotebookLM (carries from v4.2 Q2
  finding); the §5.16.5 "60s" default-value prose discrepancy is
  still an inherited DASH 6th oddity (carries from v4.2 Q1
  finding); and the N-1 remedy variants are still three
  (Period-wrapper recommended) carrying from v4.1.
- If/when the WG resolves OQ-4 (`DynamicPresentation` event) or
  OQ-5 (concurrent overlay carrier shape), the conformance audit
  will need a fresh NotebookLM query against DASH 6th to verify
  the new construct's placement and ignore-if-unknown semantics.
  v7 does not require that query because v7 carries no new
  construct.

## Marginal items detail

### M-1. `<svta:RenderableAsset>` child `<EventStream>` reinterprets baseline timing semantics (C-11)

**Carry-forward from v5 M-1**. Construct, placement, and
ambiguity unchanged in v5.1.

v5.1 §5.5.4 added a single-rooted beacon-set tightening (closes
v5 A-1) clarifying that when both the sub-MPD callback carrier
and the inline `<svta:RenderableAsset>` carrier are present on a
video form, the inline carrier wins and the sub-MPD events are
suppressed for that rendering instance. This is a Player-side
firing-policy clarification — useful for implementer
auditability, but it does NOT change the underlying conformance
question, which is the pattern of placing baseline DASH
`<EventStream>` inside an SGAI-namespaced parent with reinterpreted
`presentationTime` semantics.

The pattern remains admissible because DR-2 / DR-3 strip the
foreign-namespace parent on legacy parsers; no legacy Player ever
observes the reinterpretation. `context/08-dash-extension-rules.md`
still lacks a normative DR-8 admitting "baseline DASH element
under SVTA parent with reinterpreted reference frame" as a
canonical pattern.

**Suggested clarification**: unchanged from v5 — either rename
the SGAI-side tracking carrier to a new SVTA-namespaced element,
or canonise the pattern in `context/08-dash-extension-rules.md`
as DR-8. v5.1 honours the prior v5 punt; tracked in Refinement
gaps row 8.
